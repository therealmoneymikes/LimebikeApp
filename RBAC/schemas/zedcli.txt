


Zed can talk to a running instance of spicedb

1. Start SpiceDB locally (dev mode)
spicedb serve --grpc-preshared-key "supersecret" --http-enabled=true --datastore-engine=memory

2. Push a schema (schema.zed)
    1. zed context set local localhost:50051 supersecret
    2. zed schema write schema.zed

2b. Create schemas for each entity, then 
    create schema.zed as the entry point that imports everything

    e.g 
    import app.zed
    import bike.zed 
    import user_roles.zed 
    import pickup.zed

3. Write the permissions to user (SpiceDB will load the whole model.)
zed schea write RBAC/schema/schema.zed 

4. Check Permissions
zed permission check document:1 read user:michael
